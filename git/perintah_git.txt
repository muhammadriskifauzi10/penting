Git adalah tools version control system yang digunakan untuk merekam / snapshot segala yang terjadi pada repository
yang kemudian dijadikan riwayat / commit untuk mempermudahkan memanggil riwayat sebelumnya jika diperlukan...

Perintah untuk membuat project folder git => "git init"
Perintah untuk melihat semua status yang terjadi dalam folder git / working directory => "git status"
Perintah untuk memasukkan file ke staging area => "git add namafile"
Perintah untuk me-restore file yang berada di dalam staging area kembali pada working directory 
								=> "git rm --cached namafile"
Perintah untuk menyimpan file ke dalam repository selamanya => "git commit -m "Pesan anda" "
Perintah untuk menampilkan riwayat repository berdasarkan commit, author dan date
								=> "git log"
Perintah untuk menampilkan riwayat repository berdasarkan commit
								=> "git log --oneline"
Perintah untuk menampilkan riwayat repository berdasarkan commit dan tanda (branch/cabang)
								=> "git log --oneline graph"
"git branch" => Perintah untuk melihat daftar list cabang
"git branch namacabang" => Perintah untuk membuat cabang baru
"git checkout namacabang" => Perintah untuk memilih cabang
"git merge namacabang" => Perintah untuk menggabungkan cabang pada saat cabang berada dalam cabang master
"git branch -d namacabang" => Perintah untuk menghapus namacabang
"git revert hash" => Perintah untuk membuat commit lama menjadi commit terbaru
"git reset hash" => Perintah untuk mereset commit yang berada diatas hash commit reset
"git commit --amend -m "Message" " => Perintah untuk mengganti pesan commit terbaru

== Note ==
Ada saatnya anda mendapatkan "merge conflict" pada saat membuat cabang masalah itu terjadi biasanya ketika anda
membuat 2 cabang lalu di dalamnya terdapat 2 nama file yang sama. Kemungkinan itu akan terjadi masalah pada 
cabang master ketika anda ingin me- "merge" nya tapi tenang aja permasalahan ini tak cukup sulit untuk
mengatasinya anda hanya tinggal memilih cabang mana yang cocok untuk file anda kedepannya.
